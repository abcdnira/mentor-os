# 01 - Core Philosophy

## 1. Mentor OS 不是聊天机器人

聊天只是入口，不是产品本体。

真正的产品本体是：

- 用户画像
- 能力图谱
- 项目图谱
- 知识图谱
- 反思系统
- 路线规划
- 长期记忆

## 2. Context 不是 Memory

大模型上下文窗口是有限的。

Mentor OS 不应该把所有聊天记录无限塞进上下文，而应该通过 Reflection 把聊天压缩为结构化长期记忆。

错误方式：

```text
把所有聊天记录保存起来，下次全部塞给模型
```

正确方式：

```text
聊天记录 → Reflection → 知识节点 / 能力节点 / 项目节点 / 计划节点
```

## 3. Memory 不是保存文字，而是提炼用户

Mentor OS 的记忆对象不是“用户说过的话”，而是：

- 用户是谁
- 用户会什么
- 用户不会什么
- 用户做过什么项目
- 用户如何思考
- 用户容易在哪里焦虑
- 用户下一步该做什么

## 4. 知识必须多视图

同一个知识点应支持不同视图：

- Markdown 长文
- Knowledge Card 知识卡片
- MindMap 思维导图
- Flash Card 复习卡片
- Graph 节点关系
- Interview Answer 面试回答

底层是同一份知识，前端只是不同展示方式。

## 5. Mentor 必须长期主义

Mentor 不只回答当前问题，还要考虑：

- 这个回答是否帮助用户三个月后的面试
- 是否帮助用户一年后的职业成长
- 是否帮助用户建立长期能力
- 是否减少用户无效焦虑
- 是否把复杂问题拆成可执行步骤

## 6. Mentor 不迎合用户

Mentor 应该支持用户，但不能盲目附和。

当用户焦虑时，Mentor 要先帮助用户翻译问题、收敛范围、建立框架，而不是继续堆更多信息。

## 7. 产品最高原则

> 所有功能都必须服务于用户长期成长。

如果一个功能只是炫技，但不能帮助用户成长，就不应该优先开发。
