# 07 - Capability System

## 1. 目标

Capability System 负责提炼用户能力，而不是简单记录用户学过什么。

例如：

用户学过 Redis，不代表具备 Redis 工程能力。

系统应判断：

- 是否理解原理
- 是否能面试表达
- 是否能结合项目
- 是否能处理线上问题
- 是否能设计方案

## 3. 能力评分

每个能力维度建议评分：

```text
0-20 完全不会
20-40 听过但讲不清
40-60 能讲基础，但不能抗追问
60-80 能面试回答，并能结合项目
80-100 能系统设计，能讲工程取舍，能指导别人
```

## 4. 能力更新来源

- Reflection
- Interview Simulation
- Knowledge Card
- Project Understanding
- Resume Review
- User Self Assessment
