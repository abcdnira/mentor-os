# 09 - Interview System

## 1. 目标

Interview System 不是普通刷题系统，而是真人导师式模拟面试。

它需要：

- 根据用户目标岗位出题
- 根据用户项目追问
- 根据用户回答动态加压
- 评分
- 找漏洞
- 生成复习卡片
- 更新能力画像

## 2. 面试模式

MVP 支持：

- Golang 八股
- Redis
- MySQL
- MQ
- IM 系统设计
- 红包/钱包系统设计
- Agent 工程
- 综合模拟

## 3. 面试流程

```text
选择模式
↓
读取用户画像
↓
读取相关项目
↓
生成面试计划
↓
出题
↓
用户回答
↓
追问
↓
评分
↓
漏洞分析
↓
生成知识卡片
↓
更新 Capability
```

## 4. 评分维度

- 准确性
- 完整性
- 底层原理
- 项目结合
- 表达清晰度
- 追问承压能力
- 系统设计能力

## 5. 输出报告

每次面试生成：

```json
{
  "score": 72,
  "strengths": ["能结合红包项目讲 Redis 分布式锁"],
  "weaknesses": ["没有讲 Lua 解锁", "没有讲锁过期风险"],
  "next_actions": ["补 Redis Lua", "练习红包防超发回答"],
  "updated_capabilities": [
    {"name": "Redis 分布式锁", "score_delta": 5}
  ]
}
```

## 6. 追问策略

追问不是随机的。

应根据用户回答缺失点追问。

例如用户回答 map 非线程安全，但没讲扩容：

> 你刚才说 map 并发读写会 panic，那 map 扩容过程中读写请求是怎么处理的？

## 7. 面试收口

每次面试结束必须生成：

- 一张知识卡片
- 一个漏洞清单
- 一个下次复习计划
